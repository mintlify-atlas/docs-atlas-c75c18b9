## Anthropic Claude

Set up Anthropic's Claude models:

### Get API Key

<Steps>
  <Step title="Create Account">
    Sign up at [console.anthropic.com](https://console.anthropic.com)
  </Step>
  <Step title="Generate Key">
    Navigate to Settings → API Keys → Create Key
  </Step>
  <Step title="Copy Key">
    Save your API key (starts with `sk-ant-`)
  </Step>
</Steps>

### Configure in Glass

<Tabs>
  <Tab title="Settings UI">
    <Steps>
      <Step title="Open Settings">
        `cmd-,` or `ctrl-,`
      </Step>
      <Step title="Navigate to AI">
        Settings → AI → Providers → Anthropic
      </Step>
      <Step title="Paste Key">
        Enter your API key and save
      </Step>
    </Steps>
  </Tab>
  
  <Tab title="JSON Config">
    Set via environment variable:
    ```bash
    export ANTHROPIC_API_KEY="sk-ant-..."
    ```
    
    Or configure custom endpoint:
    ```json
    {
      "language_models": {
        "anthropic": {
          "api_url": "https://api.anthropic.com"
        }
      }
    }
    ```
  </Tab>
</Tabs>

### Available Models

<Tabs>
  <Tab title="Production">
    | Model | Context | Best For |
    |-------|---------|----------|
    | claude-4.6-sonnet | 200k | General use, balanced performance |
    | claude-4.6-opus | 200k | Complex reasoning, highest quality |
    | claude-4.6-sonnet-thinking | 200k | Tasks requiring reasoning steps |
  </Tab>
  
  <Tab title="Legacy">
    | Model | Context | Notes |
    |-------|---------|-------|
    | claude-3.7-sonnet | 200k | Previous generation |
    | claude-3-opus | 200k | Previous generation |
    | claude-3-haiku | 200k | Fast, cost-effective |
  </Tab>
</Tabs>

### Custom Models

Add custom Anthropic models:

```json
{
  "language_models": {
    "anthropic": {
      "available_models": [
        {
          "name": "claude-custom-model",
          "display_name": "Claude Custom",
          "max_tokens": 200000,
          "max_output_tokens": 8192,
          "cache_configuration": {
            "max_cache_anchors": 4,
            "should_speculate": true,
            "min_total_token": 1024
          }
        }
      ]
    }
  }
}
```

## OpenAI

Configure OpenAI models:

### Get API Key

<Steps>
  <Step title="Create Account">
    Sign up at [platform.openai.com](https://platform.openai.com)
  </Step>
  <Step title="Add Payment">
    Add payment method (required for API access)
  </Step>
  <Step title="Generate Key">
    Navigate to API Keys → Create new secret key
  </Step>
  <Step title="Copy Key">
    Save your API key (starts with `sk-`)
  </Step>
</Steps>

<Warning>
  OpenAI API access requires a paid account with credits. GitHub Copilot subscription does not include API access.
</Warning>

### Configure in Glass

<Tabs>
  <Tab title="Settings UI">
    <Steps>
      <Step title="Open Settings">
        `cmd-,` or `ctrl-,`
      </Step>
      <Step title="Navigate to AI">
        Settings → AI → Providers → OpenAI
      </Step>
      <Step title="Paste Key">
        Enter your API key and save
      </Step>
    </Steps>
  </Tab>
  
  <Tab title="Environment">
    Set environment variable:
    ```bash
    export OPENAI_API_KEY="sk-..."
    ```

  <Tab title="JSON Config">

## Ollama

Run models locally with Ollama:

### Setup Ollama

<Steps>
  <Step title="Install Ollama">
    Download from [ollama.ai](https://ollama.ai):
    ```bash
    # macOS
    brew install ollama
    
    # Linux
    curl -fsSL https://ollama.ai/install.sh | sh
    
    # Windows
    # Download installer from ollama.ai
    ```
  </Step>
  
  <Step title="Start Ollama">
    ```bash
    ollama serve
    ```
    
    Or use system service (Linux/macOS):
    ```bash
    systemctl start ollama
    ```
  </Step>
  
  <Step title="Pull Models">
    Download code models:
    ```bash
    # For agents
    ollama pull qwen2.5-coder:32b
    
    # For completions
    ollama pull codellama:7b
    
    # For chat
    ollama pull llama3.3:70b
    ```
  </Step>
  
  <Step title="Configure Glass">
    ```json
    {
      "language_models": {
        "ollama": {
          "api_url": "http://localhost:11434"
        }
      },
      "agent": {
        "default_model": {
          "provider": "ollama",
          "model": "qwen2.5-coder:32b"
        }
      }
    }
    ```
  </Step>
</Steps>

## OpenAI-Compatible APIs

Connect to any OpenAI-compatible endpoint:

### Configuration

```json
{
  "language_models": {
    "openai_compatible": {
      "api_url": "https://your-endpoint.com/v1",
      "api_key": "your-key",
      "available_models": [
        {
          "provider": "custom",
          "name": "your-model",
          "max_tokens": 8192
        }
      ]
    }
  }
}
```

## Google Gemini

Access Gemini via Agent Servers:

### Configuration

```json
{
  "agent_servers": {
    "gemini": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-gemini"],
      "env": {
        "GEMINI_API_KEY": "your-gemini-key"
      }
    }
  }
}
```

### Get API Key

<Steps>
  <Step title="Create Account">
    Sign up at [makersuite.google.com](https://makersuite.google.com)
  </Step>
  <Step title="Generate Key">
    Navigate to Get API Key → Create API Key
  </Step>
  <Step title="Configure">
    Add key to agent server config
  </Step>
</Steps>

## Provider Selection

Choose the right provider for your needs:

### Decision Matrix

| Priority | Recommended Provider | Why |
|----------|---------------------|-----|
| Best quality | Anthropic Claude | Industry-leading models |
| Latest models | OpenAI | Cutting-edge capabilities |
| Privacy | Ollama | Local, offline, private |
| Cost | Ollama | Free, unlimited |
| Speed | OpenAI GPT-4o | Fastest cloud models |
| Customization | OpenAI-compatible | Self-hosted, flexible |

### Multi-Provider Setup

Use multiple providers simultaneously:

```json
{
  "agent": {
    "default_model": {
      "provider": "anthropic",
      "model": "claude-4.6-sonnet"
    },
    "inline_assistant_model": {
      "provider": "openai",
      "model": "gpt-4o-mini"
    },
    "thread_summary_model": {
      "provider": "ollama",
      "model": "llama3.3:70b"
    }
  },
  "edit_predictions": {
    "provider": "ollama",
    "ollama": {
      "model": "codellama:7b"
    }
  }
}
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="API key not working">
    - Verify key is correct (check for typos)
    - Ensure key has not expired
    - Check account has credits/subscription
    - Try regenerating the key
    - Verify API URL is correct
  </Accordion>
  
  <Accordion title="Models not appearing">
    - Check provider is authenticated
    - Verify API endpoint is reachable
    - Ensure provider is enabled in settings
    - Try refreshing Glass (`cmd-r` / `ctrl-r`)
  </Accordion>
  
  <Accordion title="Ollama connection failed">
    - Verify Ollama is running: `ollama list`
    - Check API URL: `http://localhost:11434`
    - Ensure models are pulled
    - Check firewall/network settings
    - Try restarting Ollama service
  </Accordion>
  
  <Accordion title="Rate limit errors">
    - Check your API usage/quota
    - Wait and retry
    - Consider upgrading plan
    - Switch to different provider temporarily
  </Accordion>
</AccordionGroup>
